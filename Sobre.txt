
-- Sobre a estrutura do projeto --

** Página Main **
A pasta assets guarda as imagens do site
A pasta components guarda os componentes presentes nas páginas
 - Dentro de components, a pasta footer é chamada no final da página inicial (Dentro de content-initial)
 - Dentro de components, a pasta modals é chamada na página inicial (Dentro de content-initial)
 - Dentro de components, a pasta navbar é chamada na página inicial e nos componentes de userpage
A pasta config guarda as configurações de autenticação do usuário no site
A pasta routes é responsável por criar as rotas para as páginas do site
A pasta service contém as configurações do axios (ferramenta para serviços rest)
A pasta styles contém as estilizações globais do site
O arquivo App.js é o arquivo principal da aplicação (Ele chama a pasta Routes)


-- Sobre as páginas --

** Página Main **
A página main tem o conteúdo da página inicial do site (Esta página é pública)
 - As pastas que começam com _ , são os componentes da página Main (página inicial)
 - Cada componente contém um arquivo js para implementação do código JSX
 - Cada componente contém um arquivo css para implementação do código de estilo
 - O arquivo js é reponsável por chamar os componentes e renderiza-los na tela


** Página NotFound **
A página notfound é renderizada quando nenhuma das rotas é encontrada
 - Contém um arquivo js para implementação do código JSX
 - Contém um arquivo css para implementação do código de estilo


** Página UserPage **
A página userpage é renderizada quando o usuário estiver logado no site (Esta página é privada)
 - As pastas que começam com _ , são os componentes da página userpage (página inicial)
 - Cada componente contém um arquivo js para implementação do código JSX
 - Cada componente contém um arquivo css para implementação do código de estilo
 - O arquivo js é reponsável por chamar os componentes e criar as rotas
